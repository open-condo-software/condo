This method allows a miniapp to retrieve the URL fragment (part after #) from the parent application's URL. 
This is particularly useful for handling deeplinks and initial navigation states when a miniapp is opened via a URL with a fragment.

## Platforms
- B2B-Web

## Return value

If successful, you will receive a JSON object of the following type:
<Tabs labels={['Return type', 'Return example']}>
    ```typescript
    type GetFragmentData = {
        fragment: string  // URL fragment without the leading #
    }
    ```
    ```json
    {
      "fragment": "next=/ticket/123&source=notification"
    }
    ```
</Tabs>

## How it works

When a miniapp is opened with a URL that contains a fragment (e.g., `https://condo.d.doma.ai/miniapps/your-app-id#next=/ticket/123`), this method allows you to access the fragment part (`next=/ticket/123`).

### Common use cases:
- Deeplinks to specific screens within a miniapp
- Passing initial navigation state or parameters
- Preserving app state across page reloads

## Usage example

```typescript jsx
import '@/styles/globals.css'
import { cookies } from 'next/headers'
import React, { useEffect } from 'react'

import bridge from '@open-condo/bridge'

import type { AppProps } from 'next/app'

export default function App ({ Component, pageProps }: AppProps): React.ReactNode {
    const [nextPath, setNextPath] = useState('')

    useEffect(() => {
        bridge.send('CondoWebAppGetFragment').then(data => {
            const params = new URLSearchParams(data.fragment)
            // For example, url https://condo.d.doma.ai/miniapps/your-app-id#next=/ticket/123. 
            // Then nextPath will be '/ticket/123'
            const nextPath = params.get('next')
            setNextPath(nextPath)
        }).catch((error) => {
            console.error(error)
        })
    }, [])

    return (
        <Component {...pageProps} />
    )
}
```

## Notes

- The fragment is automatically URL-decoded when retrieved
- If there is no fragment in the URL, an empty string is returned
- The fragment is preserved when the miniapp is reloaded

## Security considerations

- Always validate and sanitize the fragment content before using it
- Be cautious when using the fragment for navigation to prevent open redirect vulnerabilities
