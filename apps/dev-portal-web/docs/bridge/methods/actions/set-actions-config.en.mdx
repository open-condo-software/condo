This method allows you to set up a list of actions that will be displayed in the main interface. Only buttons are supported.
## Platforms
- B2B-Web

## Method params

| Parameter name | Parameter type | Required | Parameter description |
|---|---|---|---|
| actions | Action[] | yes | List of actions to render |

### Action (button) params

| Parameter name | Parameter type | Required | Parameter description |
|---|---|---|---|
| label | string | no | Button text |
| disabled | boolean | no | Whether the button is disabled |
| loading | boolean | no | Whether the button shows loading state |
| icon | string | no | Optional icon name (e.g., 'download') |

<Alert type='info'>
    Icon names are listed in the [UI Icons catalog](https://condo.d.doma.ai/ui/index.html?path=/story/icons--icons).
    For the Download icon, use `icon: 'download'`.
</Alert>

## Return value

If successful, the method will return a JSON object containing a list of action IDs:
```json
{ "actionsIds": ["d21ec5e9-aafe-4552-b8ce-825f9c48c7ea", "80eed388-3c10-440a-8683-74e33f0cabed"] }
```

The host generates internal action ids and returns them in the same order as the input list.
When a user clicks a button, the host emits `CondoWebAppSendActionId` as a side effect.

## Usage example

```typescript jsx
import React, { useEffect, useMemo, useState } from 'react'

import bridge from '@open-condo/bridge'
import { useSetActionHandlers } from '@open-condo/miniapp-utils'

export default function MiniappPage (): React.ReactNode {
    const [actionsIds, setActionsIds] = useState<string[]>([])
    const actions = useMemo(() => ([
        { label: 'Save' },
        { label: 'Download', icon: 'download' },
    ]), [])

    useEffect(() => {
        bridge.send('CondoWebAppSetActionsConfig', { actions }).then((data) => {
            setActionsIds(data.actionsIds)
        })
    }, [actions])

    const handlers = useMemo(() => {
        if (!actionsIds.length) return {}

        const [saveId, downloadId] = actionsIds
        return {
            [saveId]: () => {
                // handle save
            },
            [downloadId]: () => {
                // handle download
            },
        }
    }, [actions, actionsIds])

    useSetActionHandlers(bridge, handlers)

    return null
}
```

<Alert type='info'>
    Keep the order of `actions` stable if you rely on index-based ids.
</Alert>

## Updating actions

To update a button, send a full updated `actions` array again:
```typescript
import bridge from '@open-condo/bridge'

bridge.send('CondoWebAppSetActionsConfig', {
    actions: [
        { label: 'Save' },
        { label: 'Download', loading: true },
    ],
})
```

## Hiding actions

To hide all actions, pass an empty array:
```typescript
import bridge from '@open-condo/bridge'

bridge.send('CondoWebAppSetActionsConfig', { actions: [] })
```
