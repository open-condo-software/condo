Этот метод позволяет настроить список действий, которые будут отображаться в основном интерфейсе. Поддерживаются только кнопки.

## Платформы
- B2B-Web

## Параметры метода

| Название параметра | Тип параметра | Обязательный | Описание |
|---|---|---|---|
| actions | Action[] | да | Список действий для отображения |

### Параметры Action (button)

| Название параметра | Тип параметра | Обязательный | Описание |
|---|---|---|---|
| key | string | да | Ваш идентификатор действия (должен быть уникальным в списке) |
| label | string | нет | Текст кнопки |
| type | 'primary' \| 'secondary' \| 'accent' | нет | Визуальный тип кнопки |
| disabled | boolean | нет | Отключить кнопку |
| loading | boolean | нет | Состояние загрузки |
| icon | string | нет | Название иконки (например, 'download') |

<Alert type='info'>
    Список названий иконок смотрите в [UI Icons catalog](https://condo.d.doma.ai/ui/index.html?path=/story/icons--icons).
    Для иконки Download используйте `icon: 'download'`.
</Alert>

## Возвращаемое значение

При успехе метод вернет JSON объект со списком идентификаторов действий:
```json
{ "actionsIds": ["d21ec5e9-aafe-4552-b8ce-825f9c48c7ea", "80eed388-3c10-440a-8683-74e33f0cabed"] }
```

Хост генерирует внутренние id и возвращает их в том же порядке, что и входной список.
Именно эти id возвращаются в событии `CondoWebAppSendActionId`.

## Пример использования

```typescript jsx
import React, { useEffect, useMemo, useState } from 'react'

import bridge from '@open-condo/bridge'
import { useSetActionHandlers } from '@open-condo/miniapp-utils'

export default function MiniappPage (): React.ReactNode {
    const [actionsIds, setActionsIds] = useState<string[]>([])
    const actions = useMemo(() => ([
        { key: 'save', label: 'Save', type: 'primary' },
        { key: 'download', label: 'Download', type: 'secondary', icon: 'download' },
    ]), [])

    useEffect(() => {
        bridge.send('CondoWebAppSetActionsConfig', { actions }).then((data) => {
            setActionsIds(data.actionsIds)
        })
    }, [actions])

    const handlers = useMemo(() => {
        if (!actionsIds.length) return {}

        const actionIdByKey = Object.fromEntries(
            actions.map((action, index) => [action.key, actionsIds[index]])
        )
        return {
            [actionIdByKey.save]: () => {
                // обработка сохранения
            },
            [actionIdByKey.download]: () => {
                // обработка скачивания
            },
        }
    }, [actions, actionsIds])

    useSetActionHandlers(bridge, handlers)

    return null
}
```

<Alert type='info'>
    Значение `key` не возвращается в событиях клика. Используйте `actionsIds`, чтобы сопоставить ключи с сгенерированными id.
</Alert>

## Обновление действий

Чтобы обновить кнопку, отправьте полный обновленный массив `actions`:
```typescript
import bridge from '@open-condo/bridge'

bridge.send('CondoWebAppSetActionsConfig', {
    actions: [
        { key: 'save', label: 'Save', type: 'primary' },
        { key: 'download', label: 'Download', type: 'secondary', loading: true },
    ],
})
```

## Скрытие действий

Чтобы скрыть все действия, передайте пустой массив:
```typescript
import bridge from '@open-condo/bridge'

bridge.send('CondoWebAppSetActionsConfig', { actions: [] })
```
